## Refresh Commands

Validate all Compose profiles:

```bash
make compose-profile-config
```

Build and inspect application image users:

```bash
make docker-runtime-evidence
```

Run full local stack CI evidence:

```bash
make compose-ci
```

`make compose-ci` validates Compose config, validates profiles, builds the
backend and frontend images, starts the full local stack, runs smoke checks,
runs streaming and observability smoke checks, captures evidence and tears the
stack down.

## Reviewer Notes

- `docker-compose.yml` is the canonical local runtime definition.
- `docker-compose.observability.yml` is a thin overlay for checklist/reviewer
  workflows that require an explicit observability Compose file.
- API runtime user is `appuser`.
- Frontend runtime uses `nginxinc/nginx-unprivileged` and user `101:101`.
- Vendor services such as PostgreSQL, Redpanda, Prometheus and Grafana remain
  local-only dependencies and are not presented as hardened production images.

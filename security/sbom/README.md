# RetailOps SBOM implementation

This folder contains the project SBOM configuration used for local evidence, GitHub Actions, Jenkins, and ECR release workflows.

## Implemented controls

| Control | Implementation |
|---|---|
| Image SBOM | `syft <image> -o spdx-json` and `syft <image> -o cyclonedx-json` |
| Repository SBOM | `anchore/sbom-action` or `syft dir:.` |
| Release evidence | `ci-cd/reports/sbom/*.spdx.json` and `ci-cd/reports/sbom/*.cyclonedx.json` |
| Signed SBOM attestation | `actions/attest@v4` with `sbom-path` in `.github/workflows/ecr-release.yml` |

## Local commands

```bash
mkdir -p ci-cd/reports/sbom

syft dir:. \
  --config security/sbom/syft.yaml \
  -o spdx-json=ci-cd/reports/sbom/retailops-repository.spdx.json

syft retailops-api:local \
  --config security/sbom/syft.yaml \
  -o spdx-json=ci-cd/reports/sbom/retailops-api.spdx.json \
  -o cyclonedx-json=ci-cd/reports/sbom/retailops-api.cyclonedx.json
```

## Evidence rule

Do not claim supply-chain hardening in the CV until the SBOM files are generated by CI or by the ECR release workflow and archived under `ci-cd/reports/sbom/`.

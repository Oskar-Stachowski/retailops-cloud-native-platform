# RetailOps Kubernetes policy-as-code

This folder implements Kubernetes policy-as-code for local validation and admission-control candidates.

| Folder | Purpose |
|---|---|
| `policy/conftest` | CI/local policy gate for Kubernetes YAML and Kustomize config |
| `policy/kyverno` | Runtime admission policy candidates for EKS clusters using Kyverno |
| `policy/gatekeeper` | Runtime admission policy candidates for OPA Gatekeeper |

## Local validation

```bash
docker run --rm \
  -v "$PWD:/project" \
  -w /project \
  openpolicyagent/conftest:v0.56.0 \
  test k8s/base k8s/overlays/dev --policy policy/conftest
```

The source-level policy gate blocks committed `Secret` manifests unless they
are explicitly marked examples. The rendered dev overlay also contains
`retailops-runtime-secrets`, which is generated by Kustomize from
`k8s/overlays/dev/secrets/runtime-secrets.env`. That `.env` file is ignored and
local-only, so the generated Secret is allowed only when Kustomize annotates it
with:

```yaml
retailops.io/generated-by: kustomize-secretGenerator
retailops.io/secret-source: uncommitted-local-env
```

This keeps the policy strict for committed Secret YAML while allowing rendered
local validation output to pass.

## Evidence
